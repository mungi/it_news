GitHub Blog는 오픈소스 프로젝트 유지관리자가 즉시 활성화할 수 있는 6가지 무료 보안 설정을 정리했습니다. 쉬운 공격 경로를 닫아 프로젝트를 의미 있게 더 어렵게 공격하도록 만드는 것이 목표입니다.

## Detail
보안 설정은 완전한 방어가 아니지만, 기본 브랜치 보호, 비밀 스캔, 의존성 알림처럼 흔한 실수를 줄이는 통제입니다. 공급망 공격이 늘어나는 상황에서 유지관리자 기본값 강화가 중요합니다.

## Why it matters
대부분의 개발 조직은 GitHub 저장소 설정이 실제 보안 기준선입니다.

## Engineering implication
조직 템플릿과 정책으로 저장소 생성 시 보안 설정을 자동 적용하고, 예외를 주기적으로 감사해야 합니다.
