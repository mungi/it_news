# ChatGPT Work 공개, 앱·파일·브라우저를 오가는 업무 에이전트로 확장

- 출처: GeekNews
- 게시: 2026-07-10 09:10 KST
- 원문: https://news.hada.io/topic?id=31281
- 원 출처: https://openai.com/index/chatgpt-for-your-most-ambitious-work/

## 요약
핵심: OpenAI의 ChatGPT Work는 Slack, Teams, Drive, SharePoint, 이메일·캘린더·CRM을 연결해 자료 조사부터 문서·시트·웹 앱 생성까지 이어가는 업무 에이전트 흐름. 볼 것: Enterprise/Edu는 접근 권한, 연결 도구, 실행 작업, 지출 한도, Auto-review를 중앙 관리 대상으로 둠.

## 상세 메모
### 무슨 일이 있었나
- 핵심: ChatGPT가 질문 답변 도구에서 앱·파일·브라우저를 오가는 업무 에이전트로 확장 공개
- 범위: 웹·모바일은 Pro, Enterprise, Edu부터 시작해 Plus와 Business로 확대 예정
- 데스크톱: 새 ChatGPT desktop app은 Mac·Windows에서 Free 포함 모든 플랜 제공 예정

### 본문 핵심 포인트
- 업무: 고객 리서치, 캠페인 브리프, 마케팅 자산, 예산 차이 분석, 회의 준비처럼 다단계 결과물 생성 지원
- 모델: 내장 Codex 기술과 GPT-5.6이 다단계 추론, 템플릿·참조 파일 기반 자료 생성, 웹·모바일·데스크톱 실행 담당
- 연결: Slack, Microsoft Teams, Google Drive, SharePoint, 이메일, 캘린더, CRM, 내부 도구를 플러그인으로 연결 가능
- 지정: 사용자는 `@앱이름`으로 필요한 업무 컨텍스트를 직접 지정하고 ChatGPT는 관련 플러그인 자동 판단 가능
- Sites: 작업물이나 아이디어를 인터랙티브 사이트·웹 앱으로 만들고 팀 또는 공개 URL로 공유하는 공개 베타 제공
- Scheduled Tasks: 한 번 실행, 반복 실행, 이벤트 기반 실행, 시간 변화 모니터링을 앱·브라우저와 함께 수행

### 왜 중요한가
- 영향: 생성형 AI가 문서 작성 보조에서 기업 SaaS와 파일 권한을 가진 장시간 실행 에이전트로 이동하는 제품 신호임
- 리스크: Slack·Drive·CRM·캘린더 권한을 연결하면 프롬프트 주입, 과도한 데이터 노출, 승인 없는 외부 공유가 운영 리스크가 됨
- 운영: 몇 시간 지속되는 작업은 중간 승인, 작업 취소, 비용 한도, 감사 로그 없이는 장애와 보안 사고 분석이 어려움

### 시사점
- 팀 액션: Enterprise AI 도입팀은 connector별 최소 권한, 외부 공유 차단, Auto-review 기준, Scheduled Task 승인 정책 정의 필요
- 플랫폼: ChatGPT Work류 업무 에이전트는 모델 평가보다 identity, DLP, audit log, spend guardrail, rollback workflow 검증이 선행 필요
